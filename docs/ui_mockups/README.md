# UI Mockups for Exercise 3

These files are static UI mockups for Miro and class presentation use. They are not connected to the Python or Streamlit code.

The actual implemented app is still the Streamlit app in `pages/3_Exercise_3.py`.

## Files

* `mockup_viewer.html`: easiest page to open for screenshots. It loads each individual mockup HTML file inside one viewer.
* `iteration_1_basic_form.html`: basic form version. This is intentionally simple and wireframe-like.
* `iteration_2_star_rating_cards.html`: improves the rating interaction using visual stars and idea cards.
* `iteration_3_comparison_dashboard.html`: improves comparison by using a dashboard and leaderboard layout.
* `final_ui_direction.html`: combines the best design choices into a polished final concept.

## How These Support Exercise 3

The DDW Exercise 3 checkoff requires more than one UI design or iteration. These mockups show a design progression:

1. Start with a basic form.
2. Improve rating input with visual stars.
3. Improve comparison with a leaderboard layout.
4. Choose a final direction that makes the sorting purpose clear.

The final implemented Streamlit app should remain separate from these static design files.

## Screenshot Viewer

Open `mockup_viewer.html` in your browser when you want to take screenshots for Miro or presentation slides. It provides tabs for each mockup and loads the individual HTML files in a large preview frame.

The viewer is not part of the final product. It exists only for presentation and Miro design evidence.
